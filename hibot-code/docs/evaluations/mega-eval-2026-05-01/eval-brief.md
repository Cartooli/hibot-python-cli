# Evaluation Brief

## Subject

**Hi Bot Code** (live at https://code.hibot.space) — a privacy-first, client-side HTML/CSS/JavaScript learning environment published under the Bored Games umbrella (`Hi Bot Code` repo).

## Core Proposition

Free in-browser web studio with live preview, visual + code editing, guided challenges, templates, exports to major platforms, glossary/learn tiers, PWA/offline support — no account required, minimal tracking posture, aimed at beginners and self-learners.

## Key Claims & Features

- 100% client-side; sandboxed preview (opaque iframe post–2.10)
- No sign-up; privacy-first / local-first analytics framing
- Live HTML/CSS/JS editing with templates, challenges, achievements
- Export paths to CodePen, JSFiddle, Replit, CodeSandbox, Glitch, Carrd, Webflow, Softr, Gist
- WCAG-oriented UX claims; self-hosted fonts (no Google Fonts CDN)
- Educational hub: learn tiers, glossary, challenges landing pages, launch essay (`why-i-built-this.html`)
- Service worker cache aligned with `VERSION` file

## Target Audience (stated or inferred)

Beginners, bootcamps, workshops, adult self-learners avoiding toolchain friction; parents/teachers (age-12 page, COPPA notice).

## Pricing/Model

Free / MIT licensed; no subscription for core product.

## Source Material

- README.md, CHANGELOG.md, VERSION (2.13.0), index.html meta + positioning
- Repo structure: static SPA + editor.html + satellite pages
- Competitive scan: CodePen, JSFiddle, CodeSandbox, W3Schools editor, StackBlitz-class tools

## Open Questions

- Brand tension: “Bored Games” domain vs “Hi Bot Code” product name
- README freshness vs CHANGELOG (historical “v2.0 October 2025” banner vs current 2.13.0)
- Broken relative links to docs moved under `docs/archive/`
