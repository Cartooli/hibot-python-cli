# Content Strategy Outline: Positioning Hi Bot Code

**Topic:** How to publicly pitch Hi Bot Code to beginners, educators, and skeptical developers.

---

## 1. Hook & headline options

- Lead with **friction removal** (“runs before your toolchain installs”).
- Alternative: **privacy + possession** (“your pages stay in your browser until you export”).
- Alternative: **craft** (pull from `why-i-built-this` — agency vs vibe-coded sameness).

## 2. Audience lanes

1. **Self-taught adult** — essay-first narrative.
2. **Teacher / workshop** — challenges URLs + offline PWA.
3. **Developer friend** — sandbox upgrade + MIT license + no login.

## 3. Preempt criticisms

| Criticism | Response angle |
|-----------|----------------|
| “Just use CodePen” | Challenges + opaque sandbox + offline + no account — different job-to-be-done. |
| “Privacy marketing” | Named architecture choices (self-hosted fonts, no analytics cookie). |
| “Too many pages” | Hub: start at editor; everything else is optional depth. |

## 4. Differentiation table (short)

- vs CodePen: beginner challenges + essay-led brand + PWA.
- vs W3Schools: modern editor UX + export bridges.

## 5. CTA ladder

1. Try editor (primary).
2. Read why we built this (emotional).
3. First challenge (pedagogy).

## 6. Distribution channels

- Reddit r/learnprogramming, Indie Hackers “Show IH,” educator newsletters.
- GitHub README as landing for dev-adjacent traffic.

## 7. Closing arc

Return to single promise: **type HTML, see it live, graduate on your terms.**
